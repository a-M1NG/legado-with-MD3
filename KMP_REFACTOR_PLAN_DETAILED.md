# Legado KMP 重构详细实施计划

## 🎯 总体目标
- **兼容性**: 100% 兼容现有书源规则 (JS/CSS/Regex)
- **一致性**: UI/UX 与原版 Android 保持高度一致 (95%+ 相似度)
- **性能**: 翻页流畅度 ≥60fps，冷启动 <2s
- **多平台**: Android, iOS, Desktop (Win/Mac/Linux), Web

---

## 📋 阶段五：高级功能与生态兼容 (预计 4-6 周)

### 5.1 书源规则完全兼容层
**目标**: 确保现有 3000+ 书源无需修改即可运行

#### 5.1.1 JS 环境 Polyfill
- [ ] 实现 `java` 对象映射 (KMP 中模拟 Java 类行为)
  - `java.time` → `kotlinx.datetime`
  - `java.util.Base64` → `kotlinx.serialization`
  - `java.net.URLEncoder` → Ktor Utils
- [ ] 注入全局变量: `baseUrl`, `result`, `cookie`, `header`
- [ ] 支持书源特有函数: `@js`, `{{}}`, `<js>` 语法糖

#### 5.1.2 规则解析增强
- [ ] 支持多重规则链: `class@text||tag@html`
- [ ] 实现 AllInOne 规则解析器
- [ ] 支持图片解密规则 (base64/URL 变换)
- [ ] 实现 JavaScript 注入执行沙箱

#### 5.1.3 书源导入/导出
- [ ] 支持 JSON/Legacy 格式导入
- [ ] 一键分享书源 (系统 ShareSheet)
- [ ] 书源市场 API 适配 (gedoor/legado 书源接口)
- [ ] 书源校验与自动修复工具

### 5.2 本地书籍深度支持
**目标**: 完美支持 TXT/EPUB 本地阅读

#### 5.2.1 TXT 解析引擎
- [ ] 智能章节分割 (正则 + 语义分析)
- [ ] 编码自动检测 (UTF-8/GBK/BIG5)
- [ ] 目录快速索引 (B-Tree 结构)
- [ ] 大文件流式读取 (>500MB 不卡顿)

#### 5.2.2 EPUB 解析引擎
- [ ] 完整 EPUB2/EPUB3 支持
- [ ] CSS 样式提取与应用
- [ ] 内嵌字体处理

#### 5.2.3 其他格式

- [ ] PDF 基础支持 (仅渲染，非重排)

### 5.3 阅读体验极致优化
**目标**: 达到或超越原生阅读体验

#### 5.3.1 排版引擎

#### 5.3.2 翻页动画
- [ ] 仿真翻页 (物理引擎 + 阴影渐变)

- [ ] 触摸手势识别 (单击/双击/长按/滑动)

#### 5.3.3 个性化设置

## 🛠️ 技术债务与风险控制

### 高风险项
| 风险点 | 影响 | 缓解措施 |
|--------|------|----------|
| JS 引擎差异导致书源失效 | 高 | 建立书源沙箱测试，提供 Polyfill 层 |
| iOS 后台限制 | 中 | 调整下载策略，使用 Background Fetch |
| EPUB 复杂样式渲染 | 中 | 降级策略，优先保证文字可读性 |
| Web 端大文件处理 | 中 | 分片加载 + IndexedDB 存储 |

### 技术债务
- [ ] 逐步替换临时 Polyfill 为正式实现
- [ ] 重构重复代码 (DRY 原则)
- [ ] 补充缺失的文档注释
- [ ] 优化数据库查询 (添加索引/预编译)
